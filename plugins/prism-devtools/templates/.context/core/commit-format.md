# Commit Message Format

Write clear, concise commit messages that describe the **why** not the **what**.

**Good examples:**
- `Add user authentication with JWT refresh`
- `Fix race condition in order processing`
- `Refactor payment gateway to support Stripe v3`

**Avoid:**
- `fix` / `update` / `changes` (too vague)
- Messages that just repeat the diff

If your project uses a ticket system, prepend the ticket ID per your team's convention.
